## 🧠 DeepSeek V4 for Copilot Chat

**パブリッシャー**: Vizards | **インストール数**: 約146,617 | **評価**: ★5.0 | **価格**: 無料 | **ライセンス**: MIT

---

### 🎯 概要

Copilot Chat のモデルピッカーに **DeepSeek V4 Pro** と **DeepSeek V4 Flash** を追加する拡張機能です。Copilot を置き換えるのではなく、**強化する**アプローチを取っています。新しい UI やサイドバーは不要で、使い慣れた Copilot Chat 内でそのまま DeepSeek モデルを選択できます。

---

### ✨ 主な機能

| 機能 | 説明 |
|------|------|
| **モデルピッカー統合** | GPT-4o や Claude と並んで DeepSeek V4 Pro / Flash が選択可能。1M トークンコンテキスト対応 |
| **透過的 Vision Proxy** | DeepSeek V4 はテキスト専用ですが、画像をチャットにドロップすると、別の Copilot モデル（Claude 等）が画像を説明し、そのテキストを DeepSeek に渡します |
| **Thinking Mode** | `reasoning_content` に完全対応。None / High / Max の3段階で推論深度を制御可能 |
| **Copilot 全機能継承** | エージェントモード、ツール呼び出し、Instructions、スキル、MCP など全てそのまま動作 |
| **安全なAPIキー管理** | APIキーは OS のキーチェーン（`SecretStorage`）に保存。`settings.json` や Git 履歴には残らない |
| **ゼロランタイム依存** | 純粋な VS Code API + Node.js 組込のみ。Python や Docker 不要 |

---

### 🚀 使い方

1. コマンドパレットから `DeepSeek: Set API Key` を実行
2. DeepSeek API キー（`sk-` で始まる）を貼り付け
3. Copilot Chat のモデルピッカーで DeepSeek V4 Pro または Flash を選択
4. あとは通常通りチャットするだけ

---

### ⚙️ 主な設定

| 設定キー | デフォルト | 説明 |
|----------|-----------|------|
| `deepseek-copilot.baseUrl` | `https://api.deepseek.com` | APIエンドポイント（セルフホスト/プロキシ用） |
| `deepseek-copilot.maxTokens` | `0`（無制限） | 最大出力トークン数 |
| `deepseek-copilot.visionModel` | 自動 | Vision Proxy に使うモデル |
| `deepseek-copilot.debugMode` | `minimal` | デバッグモード（minimal / metadata / verbose） |

## 📧 Balance Alert Settings（残高アラート設定）

この画面でできることはシンプルで、**「残高が一定額を下回ったらメールでお知らせする」**という通知機能の設定です。自動でチャージされるわけではありません。

### 各項目の意味

| 項目 | 説明 |
|------|------|
| **CNY Alert switch** | 人民元（CNY）残高のアラートON/OFF（現在OFF） |
| **CNY Alert threshold** | 残高が ¥いくらを下回ったら通知するか（¥1〜¥5,000,000） |
| **USD Alert switch** | 米ドル（USD）残高のアラートON/OFF（現在OFF） |
| **USD Alert threshold** | 残高が $いくらを下回ったら通知するか（$1〜$5,000,000） |

> 両方ONにした場合、**両方**のしきい値を下回ったときに通知が来ます（AND条件）。

## ⚠️ 注意点・制限（画像以外）

### 1. `reasoning_content` のマルチターン問題

さっきのテストで出た警告です：

> `reasoningContent is not supported in multi-turn conversations`

Claude エンジニアは Akira から **agent-as-tool** で1回呼ばれるだけですが、その内部で Brave MCP → GPT税理士 → Gemini と **複数ターンのツール呼び出し** が発生します。Thinking mode の `reasoning_content` が2ターン目以降で欠落する可能性があります。`thinking: None` にすれば回避できますが、推論品質は下がります。

### 2. ツール呼び出しの引数スキーマ

DeepSeek V4 は tool calling に対応していますが、Strands の内部では OpenAI 互換 API として扱われます。テストでは正常でしたが、main.py の複雑なツール（`publish_file_to_site` や `ask_gpt_tax_advisor` などネストされた agent-as-tool）で想定外の挙動が出る可能性はゼロではありません。

### 3. コスト計算の修正が必要

settings.py の `MODEL_PRICING_USD` に DeepSeek V4 Pro の価格を追加する必要があります（予算ガードが動かなくなるため）。V4 Pro はキャッシュヒット/ミスで価格が変わるので、やや複雑です。

### 4. キャッシュ戦略の違い

Claude は prompt caching を Strands が自動管理しますが、DeepSeek の context caching は異なる仕組みです。`DEBUG_TOOL_LOGGING` のキャッシュヒット率ログは期待通り出ないかもしれません。

### 5. システムプロンプトの長さ

prompts.py の `CLAUDE_ENGINEER_PROMPT` はかなり長いはず。DeepSeek V4 Pro は 1M トークンなので問題ないですが、LiteLLM 経由だとシステムプロンプトの扱いが若干異なる場合があります。
